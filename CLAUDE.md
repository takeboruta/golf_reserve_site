# Project Rules for Claude

## テスト・仕様更新の標準フロー

仕様変更・不具合修正・機能追加時は、必ず以下の順で実施する。

1. `docs/test-plan.md` を更新（対象範囲・観点・未カバー）
2. `docs/` と `README.md` の仕様整合を確認し、差分を修正
3. 過去分の不足テストを追加（既存の未カバー分岐・例外系）
4. 変更分のテストを追加/修正（正常系・異常系）
5. `npm test` -> `npm run lint` -> `npm run build` を順に実行
6. PR説明に実施結果（成功/失敗、未実施理由）を記録

補足:
- 途中で失敗した場合は、修正後に失敗ステップから再実行する。
- 仕様書更新とテスト更新は、原則同一PRに含める。
